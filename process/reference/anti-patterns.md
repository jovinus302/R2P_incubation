# Anti-Patterns — 기술데모 위장 냄새 + 자동 kill 신호

S2 플래그, S3 게이트, S4 레드팀 공용.

## 기술데모 위장 냄새
- 설명에서 기술 이름을 빼면 남는 문장이 없다 (G1 위반)
- "타깃 유저: 모두"
- 데모 시나리오가 실사용 시나리오와 다르다 (연출된 입력에서만 동작)
- 핵심 가치가 "정확도 n% 향상" 같은 지표 — 사용자 결과가 아님
- 코어 루프가 유저에게 노동을 시킴 (데이터 입력, 설정, 훈련)
- 경쟁 제품 질문에 "우리 기술이 더 좋다"로만 답함 — 경험 차별점 없음

## 발산 품질 냄새 — "대학생 해커톤 수준" (round 2 중간 교훈)

아래 패턴이면 발산 재실행:
- 기존 앱에 AI 한 스푼: "X 앱인데 AI가 도와줌" 구조 (서류 도우미, 번역 도우미, ~코치, ~요약기)
- 문제-해결 1:1 매핑만 있고 관점 없음 — "편해짐" 이상의 주장이 없음
- 페르소나가 곧 시장 조사 대상 1명 (라이더, 유학생...) — 뾰족함이 아니라 좁음
- 기준: **전문 엔지니어 + UX 디자이너 팀이 모였을 때 나올 파급력** — "이건 우리 아니면 못 만든다 + 세상이 반응한다"(NotebookLM 순간). 데모를 본 사람이 신기해하는 게 아니라 불안해져야 함 (판이 바뀌는 신호).

## 자동 kill 신호
- G1/G2 게이트 실패 (rubric 참조)
- cold-start NO인데 데모 우회책도 데이터 확보 전략도 없음 (round 1 재발)
- 핵심 기능이 현재 모델이 못 하는 것에 대한 희망에 의존
- 2주 MVP 안에 데이터 파이프라인/인프라 구축 포함
- 규제·안전 이슈가 MVP 단계부터 치명적 (의료 진단 단정, 미성년 대상 등)

## round 1 사후 교훈 (실측)
C-DOT, Pages, Wellness Coach 셋 다 데모까지 갔으나 소장 평가: "모두 data가 필요하다."
→ 데이터 의존성은 브리프 섹션 7 + 루브릭 감점 + 발산 렌즈 3번으로 3중 방어.
