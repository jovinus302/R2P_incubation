# CTO Review Interface — 다운스트림 계약

파이프라인의 종착 산출물이 소비되는 방식. ideation 스코프 밖이지만 브리프가 이것을 위해 존재.

## CTO 미팅이 소비하는 것
1. `40-run-summary.md` (1페이지) — 미팅에서 여는 문서
2. `briefs/*.md` (2–3개) — 사전 읽기 또는 미팅 중 딥다이브
3. 근거 트레일 (digest, scoring) — 질문 나올 때만

## 미팅 결정
브리프별 3택: **2주 MVP 승인 / 보류 / kill**. 승인 시 즉시 확정: 엔지니어 배정(1–2명), 데모 날짜(2주 후), MVP 성공·kill 기준 (브리프 섹션 11 그대로).

## 2주 MVP 단계 (참고)
- 이때 `codex:codex-rescue`가 처음 투입됨 (ideation에서는 미사용)
- 데모 형식: 라이브, 브리프의 "데모 순간 1개" 중심, 스크립트 3불릿
- 주 1 체크: kill 기준 발동 여부

## Go/No-Go (데모 후)
- go: 정식 incubation 트랙 (팀 확대, 다음 마일스톤)
- no-go: run 폴더에 사유 기록 — 다음 라운드 캘리브레이션 데이터가 됨 (round 1 소장 피드백이 루브릭에 반영된 것처럼)
