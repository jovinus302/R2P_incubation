### Idea 1: 실시간 서류작성 동행 코파일럿  `[SAFE]`
- **문제**: "관공서·보험·비자 서식이 너무 복잡해서 화면만 보면 막막하다."
- **타깃 유저**: 국내 거주 외국인 노동자·유학생, 디지털 문해력 낮은 중장년층
- **제품 컨셉**: 화면을 실시간 공유하면 AI가 보면서 "이 칸엔 여권번호" 식으로 한 칸씩 음성 동행. 오입력은 즉시 음성으로 정정 유도, 다국어 지원.
- **why this tech now**: 실시간 화면+음성 동시 이해가 처음으로 GA 수준 안정화됨
- **근접 제품 + 차별점**: 녹화 후 챗봇 안내 서비스 — 실시간 반응형 동행이 아님. 본 제품은 "지금 이 칸"을 실시간으로 짚음
- **cold-start 데모**: YES — 화면 공유만으로 즉시 시연, 사전 데이터 불필요
- 렌즈: anti-앱 · 새 유저 세그먼트

### Idea 2: 미팅 3분 프리브리핑 (워치)  `[SAFE]`
- **문제**: "다음 미팅 상대가 누군지, 무슨 얘기가 오갔는지 회의실 들어가기 직전에야 생각난다."
- **타깃 유저**: 백투백 미팅이 많은 직장인 (매니저·세일즈)
- **제품 컨셉**: 워치가 미팅 3분 전 진동, 오늘 캘린더 항목·최근 메시지·관련 문서를 종합해 3줄 브리핑을 화면에 표시. 장기 축적 아님, 이번 미팅 건만 즉석 종합.
- **why this tech now**: 기기 내 흩어진 맥락(메시지·캘린더·앱활동)을 정합적으로 종합하는 추론이 이제 실용 수준
- **근접 제품 + 차별점**: 삼성 Now Brief/Now Nudge는 이미 유사 컨셉 출시 — 본 아이디어는 "미팅 단위 즉석 종합"에 특화, Wellness Coach류 장기 건강데이터 의존과 무관
- **cold-start 데모**: PARTIAL — 오늘 캘린더 일정 1건 + 연락처 정보만 있으면 시연 가능(과거 축적 불요)
- 렌즈: proactive 전환 · 삼성 자산 레버리지

### Idea 3: 초행 여행자용 즉석 동선 생성기 (PlaceLM 결함 수정판)  `[STRETCH]`
- **문제**: "이 도시 처음 왔는데 뭘 봐야 할지, 남이 추천한 것 말고 지금 상황에 맞는 동선이 필요하다."
- **타깃 유저**: 저장 장소가 하나도 없는 초행 여행자 (PlaceLM 대상인 "저장 헤비유저"와 정반대 페르소나)
- **제품 컨셉**: "오늘 오후 반나절, 걸어서" 라고 말하면 에이전트가 그 자리에서 리뷰·블로그·지도를 실시간 브라우징해 동선·순서·이유를 즉석 구성. 개인 저장 이력 불필요.
- **why this tech now**: 컴퓨터/브라우저 조작 에이전트 + 장문 컨텍스트 종합이 처음으로 실시간 세션 내에서 신뢰 가능한 수준
- **근접 제품 + 차별점**: Ask Maps는 신규 추천형이지 즉석 동선 복원이 아님. PlaceLM은 "내 저장 이력" 의존형(cold-start 취약) — 본 아이디어는 이력 없는 유저를 위해 라이브 리서치로 대체
- **cold-start 데모**: YES — 위치+한 줄 요청만으로 시연, 저장 이력 불필요
- 렌즈: frontier-AI-native · 개인 축적물 재활용(역발상 — 축적 우회 설계)

### Idea 4: 제안-승인형 장보기 에이전트  `[STRETCH]`
- **문제**: "매주 장보기 리스트 짜고 여러 앱 돌아다니며 담는 게 번거롭다."
- **타깃 유저**: 맞벌이 육아 가구
- **제품 컨셉**: 음성으로 이번 주 식단·알레르기를 말하면 브라우저 에이전트가 여러 장보기 사이트에서 장바구니를 채우고, 결제는 사용자가 최종 한 번에 승인.
- **why this tech now**: 브라우저 조작 에이전트가 OSWorld 83.5%급으로 임의 사이트 UI를 인간 근접 수준으로 처리 가능해짐
- **근접 제품 + 차별점**: OpenAI Instant Checkout은 완전 자율 결제로 초기 실패 후 전략 선회 — 본 제품은 처음부터 "제안 후 승인"만 설계, 완전 자율 실행 배제
- **cold-start 데모**: PARTIAL — 세션 시작 시 음성으로 말한 선호/알레르기 한 번만 있으면 됨(장기 축적 아님)
- 렌즈: 워크플로우 임베드 · 10x 싸게

### Idea 5: 즉석 계약서 다이제스트 & 비교관  `[SAFE]`
- **문제**: "전세 계약서·보험 약관이 너무 길어서 뭐가 불리한 조항인지 나 혼자 못 찾는다."
- **타깃 유저**: 첫 전세/월세 계약자, 첫 보험 가입자
- **제품 컨셉**: 계약서 전체를 업로드하면 그 자리에서 위험 조항을 짚고 쉬운 말로 설명, 표준 약관과 대조. 1회성 문서 하나로 완결.
- **why this tech now**: 1M 컨텍스트 GA로 계약서 전문을 정합적으로 붙잡을 수 있으나, context rot(실효 ~130K부터 불안정)을 고려해 문서 길이 관리가 핵심 전제
- **근접 제품 + 차별점**: NotebookLM은 소스 기반 리서치용이지 실시간 개인 상황(이 계약이 나에게 불리한가) 반영 없음
- **cold-start 데모**: YES — 업로드한 문서 1건만 있으면 시연 완결
- 렌즈: 10x 싸게 · frontier-AI-native

### Idea 6: 즉석 인테리어 영상 프리뷰 + 장바구니 에이전트  `[STRETCH]`
- **문제**: "가구 바꾸면 방이 어떻게 보일지 상상이 안 되고, 어디서 사야 할지도 모르겠다."
- **타깃 유저**: 원룸/소형 아파트 세입자
- **제품 컨셉**: 폰 카메라로 방을 비추면 몇 초 안에 재배치된 방의 영상 프리뷰를 생성하고, 마음에 들면 에이전트가 실제 가구 사이트에서 해당 아이템 장바구니를 채워 제시(결제는 승인 필요).
- **why this tech now**: 대사·립싱크 수준 정합성을 가진 영상 생성이 실사 공간에도 적용 가능한 성숙도(중간 안정성)에 도달
- **근접 제품 + 차별점**: 기존 3D 인테리어 툴은 정적 렌더링에 전문 조작 필요 — 본 제품은 카메라 한 번으로 즉석 동영상 프리뷰, 별도 학습 불요
- **cold-start 데모**: YES — 방 한 번 비추는 것만으로 시연, 이력 불필요
- 렌즈: 불가능했던 조합 · consumer↔prosumer 뒤집기

### Idea 7: 발표 코치 워치 햅틱  `[SAFE]`
- **문제**: "발표 연습할 때 내 말투·자세가 어떤지 스스로는 안 보인다."
- **타깃 유저**: 발표 준비하는 학생·프로 스피커
- **제품 컨셉**: 연습 중 폰 카메라+워치 마이크로 실시간 관찰, 말 속도·시선·군더더기 표현을 워치 햅틱 신호(예: 너무 빠름 진동)로 즉시 코칭.
- **why this tech now**: 화면+음성 동시 실시간 이해가 낮은 지연으로 안정 동작하는 최초 시점
- **근접 제품 + 차별점**: 기존 발표 앱은 말 속도·필러워드 카운트 정도 — 본 제품은 자세·시선까지 종합해 실시간 햅틱으로 피드백
- **cold-start 데모**: YES — 연습 세션 1회 라이브만으로 시연, 이력 불필요
- 렌즈: 삼성 자산 레버리지(워치) · anti-앱

### Idea 8: 반려동물 행동 통역기  `[WILD]`
- **문제**: "우리 강아지가 왜 저러는지, 지금 불안한 건지 아픈 건지 모르겠다."
- **타깃 유저**: 초보 반려동물 보호자
- **제품 컨셉**: 폰 카메라가 반려동물을 비추면 실시간으로 행동을 관찰해 "지금 분리불안 신호 보임" 식 자연어 설명, 워치로 상황별 알림.
- **why this tech now**: 영상+음성 동시 이해가 비언어적 신호 해석까지 일반화되는 초기 단계 — 미검증 확장이라 WILD
- **근접 제품 + 차별점**: 기존 펫캠 앱은 단순 행동 분류 태그만 제공 — 본 제품은 원인·맥락까지 서술형으로 설명
- **cold-start 데모**: YES — 라이브 카메라 하나로 시연, 이력 불필요 (단 정확도 검증은 별도 리스크)
- 렌즈: 새 유저 세그먼트 · 불가능했던 조합

### Idea 9: 국경간 소상공인 실시간 협상 위스퍼러  `[STRETCH]`
- **문제**: "해외 바이어랑 통화하면서 협상 타이밍·문구를 실시간으로 판단하기 어렵다."
- **타깃 유저**: 해외 플랫폼에 입점한 1인 셀러/소상공인
- **제품 컨셉**: 통화 음성 + 상품 리스팅 화면을 동시에 보며 실시간으로 번역과 함께 "지금 할인폭 언급 타이밍" 같은 귓속말 코칭 제공.
- **why this tech now**: 실시간 멀티모달(음성+화면)이 협상처럼 맥락 의존적 판단까지 지원 가능한 최초 시점
- **근접 제품 + 차별점**: 기존 번역 앱은 단어 변환뿐 — 본 제품은 협상 전략까지 실시간 코칭
- **cold-start 데모**: YES — 라이브 통화+화면 공유만으로 시연, 이력 불필요
- 렌즈: consumer↔prosumer 뒤집기 · 새 유저 세그먼트

### Idea 10: 수어-음성 양방향 실시간 통역 워치  `[WILD]`
- **문제**: "청각장애 동료와 즉석 대화할 때마다 통역 인력을 구하기 어렵다."
- **타깃 유저**: 청각장애인 및 그 동료(직장·학교)
- **제품 컨셉**: 폰 카메라가 수어를 실시간 인식해 음성으로 변환, 상대 음성은 워치 화면에 텍스트/아바타 수어로 표시. 양방향 동시 대화 지원.
- **why this tech now**: 연속 수어 인식+실시간 음성 생성 조합은 성숙도 미검증 — WILD
- **근접 제품 + 차별점**: 기존 수어 인식기는 고정 어휘·정지 동작 수준 — 본 제품은 연속 자연 대화 지향
- **cold-start 데모**: YES — 라이브 카메라만으로 시연 가능(인식 정확도는 실사용 검증 필요)
- 렌즈: 새 유저 세그먼트 · anti-앱

### Idea 11: 즉석 맞춤 축하곡 생성기  `[STRETCH]`
- **문제**: "생일·기념일에 성의 있는 걸 주고 싶은데 매번 같은 선물이다."

- **why this tech now**: 음악 생성이 가사·스타일 개인화까지 가능. Warner×Suno 라이선스 제약으로 다운로드·상업 사용 불가 — 앱 내 재생으로 회피
- **근접 제품 + 차별점**: Suno류는 범용 생성 도구 — 본 제품은 "한 사람을 위한 즉석 선물" 경험 특화
- **cold-start 데모**: YES — 한 줄 프롬프트만으로 시연, 이력 불필요
- 렌즈: 10x 싸게 · 불가능했던 조합

### Idea 12: 배달 라이더용 핸즈프리 고객응대 에이전트  `[SAFE]`
- **문제**: "배달 중 고객 메시지 오면 운전하면서 답장하기 위험하고 번거롭다."
- **타깃 유저**: 개인 배달/퀵서비스 라이더
- **제품 컨셉**: 배달 앱 화면의 현재 주문 정보를 보고 있다가 고객 메시지가 오면 워치가 음성으로 읽어주고, 라이더는 음성으로 답만 하면 에이전트가 적절한 문구로 다듬어 전송(승인 후).
- **why this tech now**: 화면 이해+음성 상호작용 결합으로 핸즈프리 맥락 응대가 가능해진 최초 시점
- **근접 제품 + 차별점**: 배달 앱 정형 답장 버튼은 맥락 반영 없음 — 현재 주문 상황 읽고 자연스러운 응대문 생성
- **cold-start 데모**: PARTIAL — 이번 주문 정보(세션 한정)만 있으면 시연, 장기 이력 불요
- 렌즈: 워크플로우 임베드 · 새 유저 세그먼트
