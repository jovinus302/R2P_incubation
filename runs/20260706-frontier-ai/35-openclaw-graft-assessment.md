# OpenClaw 컨테이너 격리 접목처 판단 (opus-architect, 2026-07-07)

질문: OpenClaw를 컨테이너 격리 전제로 어디에 접목하면 파급력이 있는가.
프레임: **격리가 오버헤드가 아니라 셀링포인트로 뒤집히는 조건** = (a) 다루는 대상이 본질적으로 무서움 (물리 액추에이션 or 개인 데이터 전량) + (b) 격리 경계가 유저에게 가시적.

## 후보 평가 요약

| # | 후보 | 파급력 | 격리=컨셉? | 판정 |
|---|---|---|---|---|
| C1 | **케이지드 버틀러** — 가전·TV·워치 플릿 물리 액추에이션, capability-gating | 자율 체이닝 + 하이재킹 라이브 봉쇄 = 불안 | 강 (목줄 서사) | **코어** |
| C2 | **볼트 에이전트** — 갤러리+지도 이력 전량 read, outbound 봉인 컨테이너 | 불안 최고치 ("나를 다 읽었네") | **최강 — 분리 불가** (Limitless 사인의 역설계) | 증폭기 (1스텝만) |
| C3 | 디지털 분신 — 플릿 cross-screen 빙의 | continuity 편의, 불안 아님 | 중 (안 보임) | C1 하부 특성 |
| C4 | 세이프 클로허브 — 샌드박스 스킬 스토어 | "안심"이지 "불안" 아님 | 최강이나 B2B 인프라 | 배제 |
| C5 | 워치 = 물리 kill-switch 앵커 | 위임 신뢰 갭 정공법 | 중 (경계의 물성화) | C1 필수 레이어로 흡수 |

## DECISION

**C1 코어 + C5 흡수 + C2 증폭기 1스텝**: "물리 세계에 손을 가진 자율 홈 에이전트 — 목줄은 네 손목에, 하이재킹은 라이브로 봉쇄."

근거:
- SwitchBot AI Hub가 OpenClaw 하드웨어 임베드 상용화 기검증 — "될까" 증명 불필요, 삼성 플릿 규모 moat 싸움으로 직행
- 격리가 무대 위 사건: ClawJacked 공격 라이브 재현 → 컨테이너가 물리 액션 차단
- 위임 신뢰 갭(자율 실행 허용 9–12%)을 워치 승인 앵커로 물성화
- 기기 상태·명령이 입력이라 데이터 파이프라인 불필요 — round 1 사인 구조 회피

## 착수 전제 3건 (미확인 — 사실 확인 후 재리뷰)

1. SmartThings 허브/Tizen TV의 리눅스 컨테이너(동급 샌드박스) 호스팅 가능 여부 — 불가면 폰/외장 허브 폴백
2. OpenClaw Gateway의 삼성 하드웨어 latency — 실시간 물리 제어 UX 성립
3. OpenClaw capability-gating이 스킬 단위 "물리 액션 하드 차단"을 실지원하는지

## 데모 리스크

- 홈오토메이션 "신기" 추락 (high): 각본을 자율 체이닝 + 하이재킹 봉쇄 두 장면 중심으로
- 봉쇄 실연이 불안을 과잉 해소 (med): "이 정도 권한을 우리가 겨우 가두고 있다" 프레이밍으로 raw power 불안 유지
