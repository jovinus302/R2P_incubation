# Idea Brief: 여행자 즉석 동선 생성기 (가칭 "TrailNow")

> **원라인 피치**: 지도 앱에 저장해둔 장소가 하나도 없어도, "오늘 오후 반나절, 걸어서" 한 줄이면 에이전트가 그 자리에서 리뷰·블로그·지도를 라이브로 훑어 근거 있는 동선을 짜준다.

## 1. 문제
"이 도시 처음 왔는데 뭘 봐야 할지, 남이 추천한 것 말고 지금 상황에 맞는 동선이 필요하다." 초행 여행자는 저장 장소도 검증 경험도 없는 상태에서 반나절 동선을 즉석으로 짜야 한다.

## 2. 타깃 유저
저장 장소가 하나도 없는 초행 여행자 — PlaceLM의 "저장 헤비유저"와 정반대 페르소나.

## 3. 제품 컨셉
1. 위치 허용 후 "오늘 오후 반나절, 걸어서" 한 줄 입력 (텍스트/음성)
2. 에이전트가 리뷰·블로그·지도를 실시간으로 훑는 과정이 짧게 노출 — 브라우징 자체가 신뢰 신호
3. 장소 3–5곳 + 도보 순서 + 선정 이유("오늘 리뷰에 이 시간대 웨이팅 적다는 언급") 동선 카드. 스와이프로 장소 교체.

## 4. Why Now
브라우저 조작 에이전트 GA급 안정화 (OSWorld 83.5%) + 실시간 멀티모달 GA + 추론 비용 $20→$0.40/1M. 기존 강자는 "저장 목록·추천 엔진" 패러다임 고착 — Ask Maps도 신규 추천형이지 라이브 동선 구성이 아님.

## 5. 시장·경쟁 스냅샷
| 제품 | 회사 | 하는 일 | 갭 |
|---|---|---|---|
| Ask Maps | | 저장·이력 반영 대화 지도 | 신규 추천형, 라이브 크로스소스 아님 |
| Wanderlog/Mindtrip/Plotline | 스타트업 | AI 여행 플래너 | 레드오션, 전부 정적 추천 |

so what: "그 자리 라이브 리서치 기반 동선"은 직접 경쟁 부재 — 단 AI 플래너 시장 자체는 붐빔.

## 6. 기술 실현성
가장 어려운 베팅: 라이브 브라우징 다건을 지연 없이 종합해 신뢰 가능한 동선으로 압축. 브라우저 조작은 최안전이나 context rot(실효 ~130K) + 관전 시간 내 일관 완료 미검증.

## 7. 데이터 의존성 ★
- 첫 wow까지 필요 데이터: **없음** — 위치 + 한 줄. cold-start **YES**.
- 계보 고백: 본 아이디어는 U1 PlaceLM의 결함 수정판. PlaceLM은 "검증한 장소의 복원"이라는 강한 POV를 갖지만 cold-start 실질 NO로 kill — 본안은 개인 축적을 라이브 웹 리서치로 대체해 함정을 원천 우회.

## 8. 차별성
Ask Maps·플래너류는 추천이지 근거 붙인 동선 재구성이 아님. moat 가설: **UX** — "라이브로 조사하는 과정이 보이는" 신뢰 연출 + 이유 달린 동선 조립. 데이터 moat 의도적 배제.

## 9. 2주 MVP 스케치
- **데모 순간**: 한 줄 요청 → 라이브 브라우징 노출 → 근거 있는 동선 카드 완성
- **cut-list**: 도시 1개(홍대/강남) 고정 + 소스 캐싱·사전 인덱싱으로 latency 완화. 예약 연동·다국어·동행 반영 스킵.
- **팀**: 엔지니어 2명 × 2주
- **데모 스크립트**: ① "오늘 오후, 홍대 3시간, 걸어서" 실시간 입력 ② 리뷰·블로그 훑는 화면 노출 ③ 이유 달린 동선 카드 + 그 자리 장소 교체 재구성

## 10. 리스크
| 리스크 | 가능성 | MVP de-risk |
|---|---|---|
| 라이브 브라우징 지연/flaky | 높음 | 부분 — 도시 1개+캐싱 |
| 결과 평범 (레드오션 대비 차별 체감 실패) | 중 | 못함 — "이유 제시" 품질이 관건 |
| episodic 리텐션 약함 | 높음 | 안 함 — 범위 밖 |
| context rot 정합성 저하 | 중 | 부분 — 소스 수 제한 |

## 11. Go/No-Go 훅
- **성공 기준**: ① 관람자가 "이유" 근거로 실사용 의사 표명 ② 요청→동선 완성이 관전 가능 시간 내 일관 완료
- **Kill 기준**: 라이브 리서치가 느리거나 결과가 기존 플래너와 다를 바 없으면 즉시 중단
- **full go 가정**: 도시 확장 시 latency 유지 / episodic에도 재사용 트리거(타 도시 방문) 실발생
- **요청 결정**: 2주 MVP 승인 — 도시 1개 한정 + latency 검증 조건
