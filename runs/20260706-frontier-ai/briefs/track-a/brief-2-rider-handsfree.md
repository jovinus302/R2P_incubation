# Idea Brief: 보이스라이더 (배달 라이더용 핸즈프리 고객응대 에이전트)

> **원라인 피치**: 라이더가 오토바이를 세우지 않고 음성만으로 고객 메시지에 자연스럽게 답장하게 해주는 워치 비서.

## 1. 문제
"배달 중에 고객이 '문 비밀번호 몇 번이에요?' 메시지를 보내면, 오토바이를 세우고 장갑을 벗고 답장을 치는 사이 신호가 바뀐다." (라이더 실인터뷰 아님 — 확인 필요)

## 2. 타깃 유저
개인 배달/퀵서비스 라이더 (오토바이, 단건 콜 수행형). 국내 규모 자릿수: 확인 필요.

## 3. 제품 컨셉
1. 배달 시작 — 앱 화면의 현재 주문 정보(주소·요청사항) 인식 상태로 대기
2. 고객 메시지 도착 — 워치가 음성으로 읽어줌
3. 라이더 짧은 음성 응답 → 에이전트가 정중한 문구로 다듬어 제시 → 승인(탭/음성) → 전송
핵심 루프: 듣기 → 음성 응답 → 승인 → 전송. 자동 전송 없음.

## 4. Why Now
- 화면 이해+실시간 음성이 처음으로 안정 실시간 대화 수준 (Gemini Live API GA, 라이브 데모 "안전" 등급)
- 위임신뢰 갭 (저위험 위임 74% vs 자율 실행 9–12%) — "승인 후 전송" 설계와 정합
- 배달 앱 정형 답장 버튼은 맥락 미반영 — 이 갭이 구조적인지 곧 흡수될 것인지는 8절.

## 5. 시장·경쟁 스냅샷
다이제스트에 라이더 응대 특화 조사 없음 — **확인 필요** (경쟁 공백, 별도 리서치 패스 권장).

| 제품 | 회사 | 하는 일 | 갭 |
|---|---|---|---|
| 정형 답장 버튼 | 각 배달 플랫폼 | 사전 정의 문구 클릭 | 맥락 미반영, 자연어 생성 없음 |

## 6. 기술 실현성
실시간 멀티모달 productizable·"안전" 등급. 가장 어려운 베팅: **주행 소음(헬멧·바람·엔진) 환경 음성 인식 정확도** — 벤치마크 부재, 1주차 실측 선행.

## 7. 데이터 의존성 ★
- 첫 wow까지: 현재 주문 세션 정보뿐. cold-start **PARTIAL** — 장기 축적 불필요.
- 데모: 가상 주문 1건 세션 시드. 실제품: 앱 화면 인식 또는 수기 입력으로 세션 시작 시 확보.

## 8. 차별성
정형 답장 대비 "맥락 반영 자연어 응대". moat 가설: UX (핸즈프리 워크플로우 임베드 + 워치).
**플랫폼 흡수 리스크 (정면 대응)**: 메시지 채널은 플랫폼 소유 — 플랫폼이 자체 LLM 응대로 업그레이드하면 흡수 가능. 실측 근거 없음(확인 필요), "단일 앱 종속 vs 워치·OS 레이어 독립성"은 opus-architect 전략 검토 플래그. 확정 가능한 대응: ① MVP 단일 앱 스코프 ② 워치 결합 UX의 검증 속도 우위 (지속 moat 여부 미검증).

## 9. 2주 MVP 스케치
- **데모 순간**: 운전 중 음성만으로 응대 완결 (청취→응답→승인→전송, 손 안 댐)
- **cut-list**: 단일 배달앱 한정, 정식 앱 연동 제외 (가상 세션 데이터)
- **팀**: 엔지니어 2명 × 2주 (음성/워치 UI 1, 문구 생성·승인 흐름 1)
- **데모 스크립트**: ① 주행 소음 섞인 오디오로 메시지 워치 재생 ② 짧은 음성 응답 → 다듬어진 문구 → 음성 승인 전송 ③ 같은 세션 두 번째 메시지에 이전 맥락 반영 응대 대비 시연

## 10. 리스크
| 리스크 | 가능성 | MVP de-risk |
|---|---|---|
| 주행 소음 음성 인식 | 중 | **함** — 1주차 판가름 |
| 플랫폼 흡수 | 중~고 | 안 함 — 전략 검토 별도 |
| 단일 앱 한정의 실사용 전환 한계 | 중 | 부분 |
| 오응대 전송 | 낮음 (승인 필수) | 함 |
| 세션 한정 데이터의 리텐션 한계 | 중 | 안 함 |

## 11. Go/No-Go 훅
- **성공 기준**: ① 소음 조건에서 핸즈프리 완결 확인 ② 시연 라이더가 "계속 써보고 싶다" 자발 요청
- **Kill 기준**: 1주차 주행 소음 음성 인식 신뢰 불가 시 중단
- **full go 가정**: 플랫폼 흡수에도 방어 지점(워치 UX·속도) 존재 / 멀티 앱 확장 시 세션 모델 유지
- **요청 결정**: 2주 MVP 승인 — 플랫폼 흡수 전략 검토 병행 조건
