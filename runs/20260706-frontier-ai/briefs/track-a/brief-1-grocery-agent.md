# Idea Brief: 제안-승인형 장보기 에이전트

> **원라인 피치**: 이번 주 식단·알레르기만 말해두면, 에이전트가 여러 장보기 사이트에서 장바구니를 채우고 결제는 한 번의 승인으로 끝낸다.

## 1. 문제
"매주 장보기 리스트 짜고 여러 앱 돌아다니며 담는 게 번거롭다." 맞벌이 육아 가구는 매주 식단을 정하고, 알레르기 등 제약을 반영하며, 여러 장보기 사이트를 오가며 장바구니를 채우는 반복 작업을 떠안고 있다.

## 2. 타깃 유저
맞벌이 육아 가구 — 식단·알레르기 제약이 있고 시간이 부족해 여러 사이트를 돌아다닐 여력이 없는 가정. 세그먼트 크기: 확인 필요.

## 3. 제품 컨셉
음성 대화창 + 승인 화면 하나.
- **1단계**: 음성으로 "이번 주 저녁 식단은 이거고, 애가 계란 알레르기 있어".
- **2단계**: 에이전트가 여러 장보기 사이트를 오가며 장바구니를 채우는 진행 상황이 실시간으로 보이고, 알레르기 품목은 자동 제외 표시.
- **3단계**: 장바구니 요약 확인 후 최종 승인 버튼 한 번으로 결제.

## 4. Why Now
브라우저 조작 에이전트가 OSWorld 83.5%(인간 근접)로 라이브 데모 "최안전" 등급 도달. OpenAI Instant Checkout은 완전 자율 결제로 초기 실패 후 전략 선회 — 자율 실행이 시기상조임을 시장이 학습. 소비자 데이터 일치: 저위험 위임 의향 74% vs 자율 구매 실행 허용 9–12% — "제안-승인" 구조가 이 신뢰 갭에 정확히 들어맞음.

## 5. 시장·경쟁 스냅샷

| 제품 | 회사 | 하는 일 | 트랙션 신호 | 우리와의 갭 |
|---|---|---|---|---|
| ChatGPT 쇼핑 개편 | OpenAI | 완전자율 실패 후 재설계 | '26-03 전략 선회 | "제안-승인" 전용 소비자 UX 부재 |
| ChatGPT Atlas / Comet | OpenAI / Perplexity | 에이전틱 브라우저 | Comet 무료화, $200M 조달 | 범용, 장보기 전용 경험 아님 |
| Manus | Butterfly Effect→Meta | 범용 자율 에이전트 | ARR $100M+/8개월, Meta 인수 | 장보기 특화 소비자 UX 없음 |

so what: 완전 자율 결제는 시장에서 이미 실패가 검증된 경로 — "승인 게이트"가 차별화이자 신뢰 갭의 안전판.

## 6. 기술 실현성
브라우저 조작은 productizable·최안전 등급. 가장 어려운 베팅: 실제 커머스 사이트(로그인 세션 유지, UI 다양성)에서 장바구니 채우기의 안정 반복 — OSWorld는 범용 태스크 기준이라 실 커머스 라이브 안정성은 확인 필요.

## 7. 데이터 의존성 ★
- 첫 wow까지 필요한 데이터: 세션 시작 시 음성 식단·알레르기 1회.
- cold-start: **PARTIAL** — 장기 축적 불필요.
- 우회: 데모는 시연자가 그 자리에서 구두 입력. 실제품의 매 세션 반복 확인 피로는 미검증 (섹션 11 가정).

## 8. 차별성
수동 멀티앱 장보기·완전 자율 결제 대비: 위임신뢰 갭에 맞춘 "제안 후 승인" 경계 설계 자체가 핵심. moat 가설: **UX(신뢰 경계 설계)** — 데이터·유통 moat 약함, 별도 검증 필요.

## 9. 2주 MVP 스케치
- **데모 순간**: 음성 입력 후 몇 분 안에 2개 사이트에서 채워진 장바구니 요약 + 알레르기 자동 제외 표시 + 승인 한 번으로 완료.
- **cut-list**: 사전 로그인 세션만 사용 (최초 로그인/가입 스킵) · 대상 사이트 2개 제한 (P1 라이브 리스크 완화 — 실사이트 로그인·봇 대응은 orchestrator 리스크 노트 기반, 원문 확인 필요) · 실결제 스킵, 모의 결제 · 알레르기 파싱은 제한 키워드 세트.
- **팀**: 엔지니어 1–2명 × 2주.
- **데모 스크립트**: ① 음성으로 식단+알레르기 발화 ② 2개 사이트 실시간 장바구니 채우기 + 알레르기 제외 표시 ③ 요약 확인 후 승인 1클릭 완료.

## 10. 리스크
| 리스크 | 가능성 | MVP de-risk |
|---|---|---|
| 실사이트 로그인/봇 탐지로 라이브 실패 | 중–높음 | 부분 — cut으로 표면 우회, 근본 미해소 |
| 매 세션 구두 확인의 사용 피로 | 중 | 아니오 — 1회성 데모론 검증 불가 |
| 위임신뢰 갭 수치의 한국 적용성 불명 | 중 | 아니오 — 다이제스트 Open Questions |
| 사이트 UI 다양성 일반화 실패 | 중 | 부분 — 2개 제한, 확장성 미검증 |

## 11. Go/No-Go 훅
- **성공 기준**: ① 관람자가 "내 계정으로 진짜 써보고 싶다" 자발 발화 ② 장바구니 요약만 보고 즉시 신뢰 판단 가능 ③ 알레르기 자동 제외 순간 즉각 긍정 반응.
- **Kill 기준**: 1주차에 2개 사이트 중 하나라도 로그인 세션 유지 장바구니 채우기가 안정적으로 안 되면 중단.
- **full go 가정**: 반복 구두 확인 피로 없음 / 신뢰 갭 수치 한국 적용 / 3개+ 사이트 확장 안정성.
- **요청 결정**: 2주 MVP 승인
