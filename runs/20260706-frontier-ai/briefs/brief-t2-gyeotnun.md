# Idea Brief: 곁눈 (실시간 다크패턴·사기 가디언)

## 1. 문제
"'무료체험' 눌렀다가 나도 모르게 매달 결제되고, 해지 버튼은 어디 숨었는지 못 찾겠다." 결제·약관·낯선 링크 화면은 다크패턴과 사기 설계가 인간의 주의력보다 빠르게 진화한 전장. 특히 부모님 세대는 이 화면 앞에서 판단을 도와줄 사람이 곁에 없다.

## 2. 타깃 유저
범용 일반인, 특히 디지털 리터러시 낮은 부모님 세대. "결제·계약·낯선 링크를 마주하는 순간"이라는 구체 접점으로 공략.

**타깃-wedge 정합 (레드팀 반영)**: 수동 공유 wedge는 "언제 공유할지 아는" 유저를 전제 — 부모님 세대와 모순. 해소: MVP 타깃은 **자각형 유저 본인** + **가족 공유 트리거** — 자녀가 부모님 폰에 설치해주고 "결제 화면 나오면 이 버튼" 단일 트리거로 좁힘. 부모님 세대 직접 자율 사용은 상시 가디언 전환(full go 이후) 단계의 목표로 명시 이관.

## 3. 제품 컨셉
위험해 보이는 화면을 실시간 공유하면 AI가 그 자리에서 "이 무료체험은 3일 뒤 월 4만원 자동결제, 해지 버튼은 하단 회색 텍스트에 숨어있음"을 짚어줌. 차단하는 백신이 아니라 옆에서 같이 봐주는 눈.

하루 시나리오: ① 설치 화면 "무료체험" 앞에서 공유 켬 → ② 3줄 요약 즉시 ("3일 후 자동결제 전환, 해지 경로 여기") → ③ 낯선 문자 링크 열기 전 재공유 → "결제 유도 + 발신자 미검증" 경고로 진입 중단.

## 4. Why Now
Gemini Live API GA — 화면 실시간 보며 대화, 라이브 데모 "안전" 등급. 온디바이스 GenAI 폰 54%→70% 전망으로 상시 화면 어웨어니스 인프라 성숙. 기존 강자(애드블록·백신)는 규칙 기반 — "처음 보는 화면"의 의도 해석 불가.

## 5. 시장·경쟁 스냅샷
| 제품 | 하는 일 | 갭 |
|---|---|---|
| 애드블록/백신류 | 알려진 시그니처 차단 | 규칙 기반 — 신규 다크패턴 해석 불가 |
| Apple Siri AI 온스크린 어웨어니스 ('26-06) | 화면 인지 비서 | 사기·다크패턴 특화 판정 미확인 — 확인 필요 |
| 실시간 다크패턴 판정 전용 제품 | — | 다이제스트상 확인 사례 없음 — **확인 필요** |

So what: 화면 인지 비서는 빅테크 동시 접근 중이나, "사기·다크패턴 실시간 판정" wedge 선점 제품은 미확인.

## 6. 기술 실현성
가장 어려운 베팅: "처음 보는 화면"에서 결제 함정·해지 경로를 낮은 지연으로 정확히 — 규칙표가 아니라 화면의 **의도** 실시간 추론. 멀티모달 GA는 준비됨. 국내외 UI 커버리지·오탐률은 확신 없음 — 확인 필요.

## 7. 데이터 의존성 ★
- 첫 wow까지 필요 데이터: **없음** — 화면 공유 자체가 입력. cold-start **YES**.

## 8. 차별성
**P1 질문 정면 대응** ("규칙 기반으로도 상당수 잡힌다 — frontier 증분?"): 처음 보는 화면(신규 다크패턴, 미등록 사기 사이트)의 의도 실시간 해석은 규칙표로 불가능 — 이게 증분. 단 **moat 얇음 인정** — 동일 API는 경쟁사도 접근 가능.
**moat 가설**: 알고리즘이 아니라 **유통** — 온디바이스 상시 화면 어웨어니스를 삼성 폰 기본 탑재, "더 똑똑한 판정"이 아니라 "항상 거기 있는 판정". MVP 단계 검증 불가, 탑재 채택 후에만 성립하는 가정임을 명시.

## 9. 2주 MVP 스케치
- **데모 순간**: 결제/약관 화면 공유 즉시 숨은 자동결제·해지 함정 3줄 적시.
- **구성 (cut-list 준수)**: ① 준비된 다크패턴 시연 사이트 1개 (안정 wow) ② 참관자가 즉석에서 고른 임의 사이트 1개 ("처음 보는 화면" 라이브 증명) 병행.
- cut-list: 국내 UI 전수 커버리지·다국어·결제사 API 연동 스킵. 판정 정확성 보증은 시연 범위 내로 한정.
- 팀: 엔지니어 2명 × 2주 (화면 캡처·판정 파이프라인 / 판정 프롬프트·경고 UX).
- 스크립트: ① 준비 사이트 "무료체험" 직전 공유 → 함정 즉시 노출 ② 참관자 임의 사이트 → 동일 속도·정확도 ③ **오탐 측정 (레드팀 반영)**: 정상 결제/약관 화면 10개 연속 통과 시연 → 오경보 건수 현장 보고 ("함정만 잡고 정상은 안 건드린다" 증명) ④ "차단이 아니라 판단" 원라인으로 각인.

## 10. 리스크
| 리스크 | 가능성 | MVP de-risk |
|---|---|---|
| **오탐** — 정상 화면을 함정으로 오판, 신뢰 붕괴 | 중~높음 | 부분 — 광범위 오탐률은 범위 밖 (명시) |
| **retention 판정콜 (P2)** — 상시 배경 가디언 전환 시 개입 시점 판정 자체가 리스크 (과소=무용, 과다=피로) | 높음 | 아니오 — MVP는 수동 공유 wedge만 검증 |
| moat 얇음 — 빅테크 즉시 모방 | 중 | 아니오 — 탑재 가설 검증 불가 |
| 판정 latency가 실결제 흐름 못 따라감 | 중 | 부분 |
| 부모님 세대의 공유 액션 진입장벽 | 중 | 아니오 |

## 11. Go/No-Go 훅
- **성공 기준**: ① 준비 사이트에서 숨은 함정 3줄 이내 정확 적시 ② 즉석 임의 사이트에서도 세션 내 실시간 유의미 판정 (허위 없이) ③ **정상 화면 10개 중 오경보 ≤1** (오탐 측정, 레드팀 반영).
- **Kill 기준**: 1주차에 즉석 사이트 판정 지연이 라이브 불가 수준이거나 오판정 반복 시 중단.
- **full go 가정**: ① 상시 배경 판정 전환에도 오탐률 사용 가능 수준 ② 삼성 폰 기본 탑재 유통 경로 성립 ③ 수동 공유→상시 가디언 전환이 판정콜 리스크 없이 가능.
- **결정 경계 (레드팀 반영)**: 이 MVP의 성공은 "수동 공유 시나리오의 판정 능력"까지만 authorize — 상시 가디언 제품의 go가 아님. full go 가정 3건은 전부 wedge 밖이므로, MVP 성공 시 결정은 "판정 능력 검증 완료 + 상시 전환 검증 트랙 착수 승인"으로 한정.
- **요청 결정**: 2주 MVP 승인
