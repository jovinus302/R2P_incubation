## 기술 설명
집 안 다중 가전(냉장고·세탁기·에어컨·TV 등 SmartThings 연결 기기)의 실시간 상태 스트림을 실리콘 수준 기기 인증과 결합해, 하나의 검증된(위조·해킹·환각 불가능한) "가정 상태 그래프"로 만드는 능력. 에이전트가 물리 세계를 읽을 때 웹 스크래핑이나 사용자 진술이 아니라 제조사가 암호학적으로 보증하는 센서 상태를 근거로 삼게 하는 레이어. 상태는 실시간 스냅샷이며 장기 축적 이력이 아니다.

## 시드 설득력 테스트 (3문) — v3 게이트

판정 주체: opus-architect P3 페르소나 (S0.5 사전 채점 겸용, `runs/_sourcing/20260707.md`).

1. **전략적 긴장**: OpenClaw류 오픈소스 홈 에이전트가 스킬 마켓으로 급확산 중이나 보안 격리 실패 사례가 이미 발생 — 삼성은 자체 실리콘 신뢰 루트로 인증된 가전 플릿을 가진 유일한 홈 에이전트 주체다, 오픈소스가 보안 사고로 신뢰를 잃기 전 창.
   - 가설: "오픈소스 홈 에이전트 신뢰 붕괴 vs 삼성 인증 가전 플릿 — 12개월 창"
2. **삼성 비대칭**: 가전 플릿 + 실리콘 기기 인증 = **필수·제조 실물**. 구글은 가전을 만들지 않고, OpenAI/스타트업은 API 레이어 접근뿐 — 제조사 수준 기기 신뢰는 구조적으로 삼성만.
   - 필수 자산: SmartThings 연결 가전 플릿, 실리콘 기기 인증(신뢰 루트)
3. **다음 문**: "검증된 물리 상태" 위의 홈 에이전트 플랫폼 — 보험(실제 상태 증명), 안전(오작동 검증), 에너지(신뢰 가능한 자동화) 제품군의 기반.
   - 다음 문: 신뢰 홈 에이전트 플랫폼 기반 레이어

**판정**: **PASS** — 근거: 세트 중 유일하게 비대칭이 유통도 데이터 축적도 아닌 제조 실물. P3 3대 질문 방어됨.
- 진입 조건: (a) 홈 상태 그래프 = 실시간 상태 (축적 이력 전제 금지 — PDE 재판), (b) 신뢰 앵커 위 killer 유스케이스 1개 구체화 (S2 몫).

**역할 분리**: 게이트는 "가전 플릿+실리콘 인증" 비대칭의 존재로 PASS — S3 moat 축은 이 존재를 재심하지 않고 각 아이디어가 이를 구조적으로 쓰는지만 채점.
